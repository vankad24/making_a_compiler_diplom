### 4. Принципы создания языков программирования
#### 4.1 Виды языков программирования по принципу исполнения

#### 4.1 Общая схема компилятора (лексический, синтаксический анализ, генерация кода)
Работа любого языка программирования начинается с считывания текста, представляющего собой исходный код программы, и последующее преобразование текста в удобную для анализа структуру.  Зачастую этой структурой является дерево разбора. Это сложный процесс, поэтому его разбивают на несколько этапов, за которые отвечают разные подпрограммы. В общем случае порядок работы современного компилятора или интерпретатора выглядит следующим образом:
Исходный код, лексический анализ (Lexer), синтаксический анализ (Parser), семантический анализ, генерация исполняемого файла / интерпретация.
Для начала начнём с лексического анализа.

#### 4.2 Лексический анализ (токенизация, регулярные выражения, конечные автоматы)
Исходный код, будь то текст из файла или просто строка, является простым набором символов. Чтобы было удобнее работать с текстом, его нужно привести в удобный вид. Самая первая мысль, которая может прийти в голову - это разбить текст на слова. Но, поскольку у нас не литературное произведение, а программный код, нужно разделять слова не только от пробелов, но и от арифметических и специальных символов. Для примера, возьмём следующий код:
`int x = 10;`
Данный код отвечает за создание целочисленной переменной `x` со значением `10`.
Разбив данную строку, мы получим:
`'int', 'x', '=', '10', ';'`
В контексте лексического анализа такие последовательности символов называют не слова, а **лексемы**.
Теперь давайте рассмотрим аналогичный фрагмент кода, который создаёт переменную с плавающей точкой:
`float num = 2.4;`
Если разбить его на лексемы, то получится:
`'float', 'num', '=', '2.4', ';'`
В этих двух фрагментах кода мы можем заметить сходство: сперва пишется тип переменной, затем название, знак равно, числовое значение, знак точка с запятой. Давайте запишем это в более общей форме:
`<тип 'float'> <название 'num'> <знак '='> <число '2.4'> <знак ';'>`
Каждый такой элемент называется **токеном**. Он представляет собой название категории символов и хранит в себе исходный текст (лексему). На практике существуют некоторые принятые названия токенов и приведённые примеры в виде набора токенов выглядели бы следующим образом:
`<KEYWORD 'int'>, <ID 'x'>, <OPERATOR '='>, <NUMBER '10'>, <KEYWORD ';'>` и
`<KEYWORD 'float'>, <ID 'num'>, <OPERATOR '='>, <NUMBER '2.4'>, <KEYWORD ';'>`.
Зарезервированные в языке символы называют KEYWORD, названия переменных или функций - ID или IDENTIFIER, арифметические знаки OPERATOR, а числовые значения - NUMBER.
Токены могут включать в себя дополнительную информацию, такую как: категорию лексемы, номер строки, имя файла.
Зачастую, чтобы разбить исходный текст на набор токенов, используют регулярные выражения (regex). Например, `IDENTIFIER` можно представить регулярным выражением `[a-zA-Z_][a-zA-Z0-9_]*`, а NUMBER представить как `[0-9]+`.

Задача лексера не только разбить текст на токены, он также отвечает за распознавание ошибок на этапе считывания. Например, в начале названия переменной недопустима цифра, поэтому ошибка в выражении `int 1x = 10;` распознается уже на этапе лексического анализа.

В современных языках программный код лексера обычно не пишут вручную. Для этого используют кодогенерацию на основе заранее прописанной лексической грамматики. Один из самых популярных генераторов лексеров - "flex", написанного на C++, от которого позже пошли аналоги для разных языков программирования: jflex, todo.
Для генерации нужно передать файл, в котором прописаны название токена и соответствующее регулярное выражение для поиска.

#### Синтаксический анализ (контекстно-свободные грамматики, деревья разбора)
Перейдём к следующему этапу преобразования нашего кода. Давайте ещё раз посмотрим на ранее упомянутые примеры:
Набор токенов для строки `int x = 10;`:
`<KEYWORD 'int'>, <ID 'x'>, <OPERATOR '='>, <NUMBER '10'>, <KEYWORD ';'>`
И для строки `float num = 2.4;`:
`<KEYWORD 'float'>, <ID 'num'>, <OPERATOR '='>, <NUMBER '2.4'>, <KEYWORD ';'>`.
Далее для удобства будем записывать `KEYWORD 'float'` как `FLOAT`, `<KEYWORD 'int'> как `INT`, `<OPERATOR '='>` как `EQUAL`, `<KEYWORD ';'>` как SEMICOLON.
Можно заметить, что полученные последовательности довольно схожи. Один из базовых подходов в программировании и в разработке программного обеспечение - это абстракция и обобщение. Здесь мы тоже воспользуемся этим методом. В общем виде инициализация переменной (variable initializaton) будет выглядеть так:
`TYPE ID EQUAL NUMBER SEMICOLON`
Тип переменной - это TYPE, название переменных, как мы уже говорили, принято называть ID, оператор присваивания - EQUAL, число для инициализации - NUMBER, а точка с запятой - SEMICOLON.
Большинство элементов здесь представляют собой простые токены, но TYPE является составным, поскольку состоит из FLOAT, либо INT. Запишем для него определение:
`TYPE: FLOAT | INT`
Знак `:` означает 'является', а знак `|` означает 'или'.
В формальных грамматиках данная запись называется правилом продукции (производства), где атомарные элементы (токены) называются терминалами, а составные нетерминалами.
Таким образом FLOAT и INT являются терминалами, а TYPE - нетерминалом.

Усложним наш пример: 
`int n = 2 + 2 - 1`
Теперь знак точки с запятой в конце является необязятельным, а вместо числа можно написать арифметическое выражение без скобок со знаками плюс или минус.
Тогда фрагмент грамматики для инициализации переменной будет выглядеть следующим образом:

```
VAR_INIT: TYPE ID ASSIGN EXPRESSION LINE_END
TYPE: FLOAT | INT
EXPRESSION: NUMBER MATH_OP EXPRESSION | NUMBER
MATH_OP: PLUS | MINUS
LINE_END: SEMICOLON |
```
VAR_INIT - это продукция инициализации переменной, TYPE - тип, EXPRESSION - арифметическое выражение, MATH_OP - математический оператор плюс или минус, LINE_END - конец строки.
В данном примере нетерминал LINE_END порождается терминалом SEMICOLON или может вовсе отсутствовать. Нетерминал EXPRESSION в своём определении имеет рекурсивный вызов, что является ключевым принципом в построении грамматик и позволяет описывать конструкции произвольной длины.




Как и код лексера, код синтаксического анализатора (парсера) генерируется на основе грамматики, описанных в таком же формате, как было описано в примерах выше. Одним из самых популярных инструментов для генерации парсера является byacc. На его основе также существуют аналоги для других языков программирования: jbyacc, todo и т.д. Таким образом, создание парсера сводится к описанию грамматики, ведь грамматика - это и есть описание языка.


Подводя итог, можно сказать следующее: синтаксический анализатор анализирует последовательность токенов, созданных лексическим анализатором, и строит структуру программы в виде дерева разбора или абстрактного синтаксического дерева (АСД). Дерево разбора подробное, по нему можно увидеть, какие правила грамматики были применены, оно может содержать такие вещи, как скобки и знаки пунктуации. Абстрактное синтаксическое дерево (АСД) - обработанное дерево разбора. В нём удалены несущественные детали, часть узлов сокращены.
